./resume
The actual resume data api is located here. This data is served with a 'Content-Type' of 'application/json;utf-8'.

API Data Access Description

The entire JSON document is navigable via the url request, so always starting with ./resume you can acess each property that is listed from the root.

Examples:

./resume — The root of the resume document.
./resume/experience — All the experience records, the employment history.
./resume/skills/1 — The second entry in the skills array.
./resume/contact — The resume contact information.

Response Package

Responses are packaged in the following data object:

{
    status: "",
    path: [],
    data: {}
}

Response Key Definitions

status => ( 'success' | 'failure' )
Indicates if the node in the resume document was found.
path => []
The scrubbed array of requested path nodes in the document.
data => ( {} | NULL )
The data requested in the path, if the status is a 'failure' then this key will be NULL.

Notes on Resume Data

  • The experience and education keys store arrays of objects with keys based on the iCalendar specification.
  • The contact key stores data based on the vCard specification.
